Other version control systems include Mercurial (also distributed) and Subversion (not distributed).

Workflow Although Git is a great tool for large-scale distributed development, for us a Subversion-like workflow with a "central repository" (where you clone/fetch from and push to) and linear history makes more sense. The instructions below assume that development is happening in a single branch. Only commit your own, original work .
